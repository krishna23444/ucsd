System Design using Datapath
and Control Subsystems

Due time: Right before class on Jun 1

Objective

In this lab, we will build a simple system using datapath, memory, control subsystems and a program counter. We use buses and standard modules as basic building blocks for datapath, memory and program counter design, and use VHDL code  for control subsystem.

Note that in this lab, we use FPGA Family: Virtex2, Device Type: XC2V80-5CS144.

The Project

Design a simple system that can perform addition, shift, comparison, and masking. The system reads a 8-bit value from the read-only memory (ROM) block (3-bit instruction set & 5-bit data). There are three 5-bit registers R1, R2, and R3. Registers R1 and R2 store the data. Register R3 presents the result of the calculations. There is also a 1-bit flag for overflow and comparison. Report

Title page:

  • Names of students and due date.
  • Title of the lab and objective.
  • A brief description of each person's contribution.

Contents:

  • Schematic diagrams of the top-level design, memory subsystem and datapath subsystem - must use bus.
  • VHDL code of the control subsystem using one truth table.
  • Layout of the design (FPGA)
  • Your instructions in the memory
  • Functional simulation of your instructions . Demonstrate the content of the output of each of the 8 instructions. Label and explain the data before and after each operation.
  • Discussion of the verification. Show the reason why we are confident that the system can function properly. List the process needed for the proof.
  • Illustrate the worst delay path between registers with timing simulation.
