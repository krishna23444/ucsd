  6. How to identify a critical path in a design ?

    A critical path is a path from an input to an output with the largest delay. Xilinx timing analysis tool provides a good way to identify critical paths. It analyses every path from the input to the output, and gives the delay on each path. We can easily find the critical path from the timing report.
    The following is an example to find a critical path in a full adder design:
    1. Choose your component
      Highlight your four-bit adder source file (e.g. four_bit_adder.sch) in the project source window of the Xilinx Project Navigator.
    2. Generate the timing report
      In the process view window, click on the ?Implement Design -> Generate Post-Place and Route Static Timing -> Post-Place and Route Static Timing Report ?step. A timing analyzer window shall pop-up with the timing report similar to the following:

    3. Identify the critical path
      The critical path is from input B to output S, with a delay of 6.428 nanoseconds, according to the timing report.

    We shall be able to get the critical path delay from simulation waveforms. Using a test bench waveform with single toggle at input B, we get the simulation wave forms as follows. Note that the exact 6.428 ns delay is shown in the waveforms.

  7. What is a four-bit adder ?
    A 4-bit adder takes as input three numbers, A , B and Ci , sum them up, and generates two output numbers, S and Co . Numbers A, B , and S are 4-bit binary numbers: A = (a3a2a1a0); B = (b3b2b1b0); S = (s3s2s1s0). Ci and Co are single bit binary numbers. A and B are two operands of the addition operation. Ci is the carry-in bit. S is the sum. Co is the carry-out bit.

  8. How to install ModelSim XE III Starter and let it work with Xilinx ISE 7.1?
    1. Download ModelSim XE III from http://www.xilinx.com/webpack/index.htm.
    2. In the installation, select ModelSim XE III Starter.
       
      Select Full VHDL as library installation type.
       
      Use default values for other options.
    3. Integrate Modelsim with Xilinx ISE. (Usually it's done by Xilinx ISE automatically. But if you didn't see the ModelSim options when you click on a Test Bench Waveform file, please integrate ModelSim manually.)
      • Click on the Edit menu in Xilinx ISE 7.1, and click on Preferences.
      • In the Preferences window, indicate the ModelSim installation directory in Integrated Tools Options.
         
      • Click OK.
